# Мультимодальная RAG по документам

## Задача

В отчётах, каталогах и сканах важная информация часто живёт **в таблицах, схемах и вёрстке страницы**, а не в «плоском» тексте. Обычный RAG по извлечённому тексту теряет контекст и даёт слабые ответы на вопросы вида «что на этой странице?», «сравни по графику».

## Зачем мультимодальный RAG

Мы ищем не по одному представлению документа, а опираемся на **страницу как изображение**: модель «видит» макет, подписи и визуальные блоки. Ответ в чате строится по **релевантным страницам**, подобранным по запросу пользователя.

## Два канала поиска

Используются **две модели эмбеддингов** (два способа понять, какая страница подходит к запросу):

| Канал | Суть |
|--------|------|
| **Текстовый** | Vision-модель (Mistral) кратко описывает страницу → эмбеддинги **BGE** → индекс **FAISS**. Хорошо для явного текста и формулировок близко к саммари. |
| **Визуальный** | **ColQwen** смотрит на пиксели страницы — сильнее там, где важна визуальная структура без нормального OCR-текста. |

В интерфейсе можно выбрать один канал или **оба сразу** (гибрид). Генерация ответа и суммаризация страниц при сборке текстового индекса — через **Mistral** (нужен API-ключ).

## Запуск в Docker

```bash
cp .env.example .env
# В .env укажите MISTRAL_API_KEY и при необходимости MODEL_NAME (см. .env.example)
./run_app.sh
```

Откройте **http://localhost:8501**. Каталог `data/` (PNG страниц + файлы индексов) монтируется в контейнер; без них UI покажет, чего не хватает.

Локальные модели поиска (BGE, ColQwen) считаются на **CPU** — первый запуск может занять много времени. Сборка и обновление индексов — скрипты в `scripts/`, пути к данным — в `src/config/*.yaml`.
